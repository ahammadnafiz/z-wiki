Three ownership zones:

- **`raw/`** — you own it. Immutable to Claude. Source material lives
  here untouched.
- **`wiki/`** — Claude owns it. Every page generated by an operation.
  `wiki/.meta/` and `wiki/indexes/` are machine-generated; never
  hand-edit.
- **`CLAUDE.md` + `.claude/`** — co-owned. You set the topic and
  policies; Claude follows them and extends via `/wiki-new-template`.

## 2. The operations

Six slash commands. Each one is a small orchestration on top of the
scripts layer. Every operation ends with the sidecar cache refreshed
and the shards in sync.

### 2.1 `/wiki-ingest`

Converts `raw/` sources into `wiki/sources/{slug}.md` summaries.
Extracts concepts and entities along the way.

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. Detect unprocessed raw files                                │
│    → read wiki/.meta/sources.json                              │
│    → diff against raw/**/*.{md,pdf,txt,docx,...}               │
│    → set of files NOT in the sidecar = to-process              │
├─────────────────────────────────────────────────────────────────┤
│ 2. Batch size policy                                            │
│    ├─ 1–2 sources   → inline in main session                   │
│    ├─ 3–10 sources  → 1 subagent per source                    │
│    └─ 11+ sources   → waves of 10, rebuild .meta between       │
├─────────────────────────────────────────────────────────────────┤
│ 3. Per-source (or per-subagent)                                 │
│    ├─ Extract (markitdown for binaries)                         │
│    ├─ Gibberish check (scripts/check_extraction.py)             │
│    ├─ Token-gate (≤50K: full read; 50–200K: chunk; >200K: stop) │
│    ├─ Write wiki/sources/{slug}.md from template                │
│    ├─ Extract concepts/entities → stubs or update existing      │
│    ├─ Flag contradictions inline                                │
│    └─ Return structured report (no raw body back to parent)     │
├─────────────────────────────────────────────────────────────────┤
│ 4. Whole-vault post-pass                                        │
│    ├─ python3 scripts/build_meta.py --embed-if-enabled          │
│    │    → rebuilds backlinks.json + sources.json +              │
│    │      search-index.tsv + (if on) embeddings.npy             │
│    ├─ Reconcile source_count/inbound_refs frontmatter           │
│    ├─ Identify stubs crossing promotion thresholds              │
│    │   → list as candidates; do NOT promote (gate is /wiki-promote)│
│    ├─ python3 scripts/shard_index.py                            │
│    │    → rebuilds wiki/index.md + per-type + tag shards        │
│    └─ Append log entries                                        │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 `/wiki-query`

Answers a question strictly from the wiki. Never invents facts.

**Retrieval ladder — never reads `wiki/index.md` first:**

```
query
  │
  ▼
1. python3 scripts/wiki_search.py "<query>"
     │  auto-selects lexical or hybrid mode
     │  returns top-N ranked paths
     ▼
2. Read each candidate page (cap ~10)
     │
     ▼
3. Is there enough to answer?
     ├─ yes → synthesize, cite, file
     └─ no  → consult wiki/indexes/by-tag/<tag>.md
                 │
                 ▼
             still no?   consult wiki/<type>/_index.md
                 │
                 ▼
             still no?   read wiki/index.md (last resort)
                 │
                 ▼
             empty?      print "No coverage." — zero writes
```

**Filing contract:**
- Output → `wiki/outputs/{question-slug}.md`
- `last_seen` bumped only on **cited** pages (not every page read)
- Sidecar + shards refreshed so next query sees the new output
- Log entry appended

### 2.3 `/wiki-lint`

Weekly health check. Tiered to stay cheap on large vaults.

```
┌──────────────────────────────────────────────────────┐
│ Tier 1 — frontmatter only (always runs)              │
│   read first ~30 lines of every page                 │
│   fix: missing fields, drifted source_count          │
├──────────────────────────────────────────────────────┤
│ Tier 2 — full body (only pages that need it)         │
│   ├─ failed Tier 1, or                               │
│   ├─ last_seen > 7 days, or                          │
│   └─ new since last lint                             │
│   fix: broken wikilinks, filename violations, dupes  │
├──────────────────────────────────────────────────────┤
│ Tier 3 — cross-page contradiction sweep              │
│   dispatched as a SUBAGENT                           │
│   reads page-pairs sharing ≥2 cited sources          │
│   returns structured report (parent never sees body) │
└──────────────────────────────────────────────────────┘
```

Promotion candidates are listed but never acted on — `/wiki-promote`
has exclusive promotion authority.

### 2.4 `/wiki-compile`

Sidecar + shard rebuild without body mutation. O(N) in page count.

```
python3 scripts/build_meta.py --embed-if-enabled
  → wiki/.meta/backlinks.json
  → wiki/.meta/sources.json
  → wiki/.meta/search-index.tsv
  → wiki/.meta/embeddings.npy         (if semantic is on)
  → wiki/.meta/embeddings-manifest.tsv

python3 scripts/shard_index.py
  → wiki/index.md                     (thin; <1K tokens)
  → wiki/{sources,concepts,entities,syntheses,outputs}/_index.md
    (auto-shards into _index-a-h.md / _index-i-p.md / _index-q-z.md
     when parent exceeds 10K-token budget)
  → wiki/indexes/by-tag/{tag}.md      (for tags with ≥3 members)
```

### 2.5 `/wiki-promote`

User-gated stub graduation. Never fires automatically.

```
/wiki-promote --list        (default)
  → print every stub with source_count≥2 OR inbound_refs≥5
  → no writes

/wiki-promote <slug>
  → show promoted draft (concept.md or entity.md template)
  → wait for user confirmation
  → write; refresh sidecar + shards; log

/wiki-promote --all
  → same flow, batched with one confirmation upfront
```

Rationale: promotion = full-page LLM generation. Previously auto-fired
in INGEST's post-pass and burned tokens on pages that weren't yet
load-bearing. Explicit gate makes cost user-controlled.

### 2.6 `/wiki-search`

Browse primitive. Returns ranked paths; writes nothing. Used as a
subroutine by `/wiki-query`.

```
python3 scripts/wiki_search.py "<query>"
  [--mode auto|lexical|semantic|hybrid]
  [--type sources|concepts|entities|syntheses|outputs]
  [--tag <tag>]
  [--limit N]
  [--json]
```

## 3. The sidecar cache — `wiki/.meta/`

Derived data. Frontmatter stays authoritative; sidecar is the fast
read. LINT reconciles drift.

| File | Shape | Purpose |
|---|---|---|
| `backlinks.json` | `{version, generated_at, nodes: {slug: {path, type, inbound_sources[], inbound_all[], source_count, inbound_refs}}}` | Inverse wikilink graph. Killed O(N²) recomputation in INGEST post-pass. |
| `sources.json` | `{version, generated_at, sources: [{slug, path, source_path, title, summary, tags, ...}]}` | Source registry. INGEST uses it to detect unprocessed raws. |
| `search-index.tsv` | One line per page: `path\ttitle\ttags\tconcepts\tsummary` | Grep substrate for `wiki_search.py`. |
| `embeddings.npy` *(optional)* | float32 `[N, 384]`, L2-normalized | Semantic vectors. |
| `embeddings-manifest.tsv` *(optional)* | `slug\tpath\ttext_sha256` | Rows align to `.npy` rows. SHA enables incremental re-embedding. |

**Write contract:**
- All writes go through `scripts/build_meta.py`.
- Writes are atomic: write to `*.tmp`, then `rename()`. Any crash
  leaves the previous version intact.
- The `--check` mode rebuilds in-memory and compares against on-disk
  frontmatter — used by `tests/test-source-count.sh` to detect drift.

**Read contract:**
- Every operation reads from here. No operation recomputes the graph
  from filesystem scan except `build_meta.py` itself.
- `scripts/semantic.py` loads `embeddings.npy` via `numpy.load` — no
  torch/sentence-transformers needed for the *query* path as long as
  vectors are precomputed (they need ST to compute the query
  embedding, so semantic search does depend on ST at query time).

## 4. The sharded index — `wiki/index.md` + `wiki/indexes/`

**Why not one big `index.md`:** at 2000 sources, a flat catalog is
~40K tokens per query. Sharding keeps per-query context bounded.

```
wiki/
├── index.md                      ← thin top-level (<1K tokens)
│   ├─ counts per type
│   ├─ links to per-type catalogs
│   └─ links to tag shards
│
├── sources/_index.md             ← type catalog
│   └─ auto-shards at 10K token cap:
│      ├─ _index-a-h.md
│      ├─ _index-i-p.md
│      └─ _index-q-z.md
│
├── concepts/_index.md            (same pattern)
├── entities/_index.md            (same pattern)
├── syntheses/_index.md           (same pattern)
├── outputs/_index.md             (same pattern)
│
└── indexes/by-tag/
    ├── transformers.md           ← auto-emitted for any tag
    ├── reinforcement-learning.md   with ≥3 members
    └── ...
```

All these are regenerated atomically by `scripts/shard_index.py` —
no partial writes, no hand-edits. Bodies of `wiki/sources/*.md` etc.
are untouched by sharding.

## 5. Search architecture

Three modes. `auto` is the default — picks `hybrid` if embeddings
exist, else `lexical`.

### 5.1 Lexical (always available)

BM25-style over `search-index.tsv`.

```
query → tokenize → stopword filter
        │
        ▼
for each entry in search-index.tsv:
    score = Σ_{token} weight[field] × tf_dampened × idf[token]

  field weights:  title=3.0   tags=2.0   concepts=2.0   summary=1.0
  tf dampening:   tf / (tf + 1.5)
  idf:            log( (N - df + 0.5) / (df + 0.5) + 1 )
```

Zero deps beyond stdlib. Python reads the TSV — the index itself is
grep-friendly for humans.

### 5.2 Semantic (optional — `/wiki-enable-semantic` to turn on)

Cosine similarity over `embeddings.npy`.

```
query → sentence-transformers (all-MiniLM-L6-v2) → normalized 384-dim vector q
        │
        ▼
scores = embeddings_matrix @ q     # dot product == cosine since normalized
        │
        ▼
top-K by score; filter by --type / --tag post-hoc
```

Model: `sentence-transformers/all-MiniLM-L6-v2` (default).
Overridable via `ZWIKI_EMBED_MODEL` env var.

### 5.3 Hybrid (default when embeddings exist)

Runs lexical + semantic in parallel, merges with Reciprocal Rank
Fusion.

```
lex_results   = lexical_search(query, raw_limit=2×limit)
sem_results   = semantic_search(query, raw_limit=2×limit)

for each ranked list:
  for (rank, entry) in enumerate(list, start=1):
    rrf_score[entry.path] += 1 / (60 + rank)   # k=60 (Cormack et al.)

merged = sort by rrf_score desc
```

RRF treats heterogeneous score scales fairly — no tuning needed.

### 5.4 Post-ranking stages

Applied to all modes after the raw ranked list:

```
1. Type diversity cap
   no single type (sources/concepts/entities/syntheses/outputs)
   exceeds 60% of top-N results
   (prevents "all 10 results are concept stubs")

2. Jaccard near-dup dedup
   for each result, compare tokens(title + summary) against kept[]
   skip if Jaccard similarity ≥ 0.85 to any already-kept result
   (prevents "same page under two different names")
```

## 6. Context engineering

Principles encoded in `.claude/skills/context-engineering/SKILL.md`
and enforced by command wording. Tied to the Anthropic team's public
guidance on context rot and the 1M window.

### 6.1 Subagent dispatch — the INGEST cost lever

INGEST of a single source reads up to 50K tokens of raw content;
only the summary + concept slugs matter to the parent session. At
batch scale, this is the single biggest cost lever.

```
batch of 3+ sources
       │
       ▼
spawn 1 subagent per source (via Agent tool)
       │
       ├─ subagent 1: reads raw, writes summary, returns report
       ├─ subagent 2:  "
       └─ subagent N:  "
       │
       ▼
parent session aggregates reports (no raw content reaches parent)
```

Parent context stays thin → scales to 20+ sources without context
rot.

### 6.2 Staleness gates — LINT's cost model

Pages with recent `last_seen` skip the full-body tier. LINT is O(N)
in pages but with tiny constant factor for the "recently read" cohort.

### 6.3 Prompt-cache friendliness

Read order: stable content first, mutable content last.
1. `CLAUDE.md` (stable; 293 lines) → cacheable prefix
2. Skill references (stable) → cacheable
3. `wiki/.meta/sources.json` or specific candidate pages (mutable) → not cached

Anthropic prompt cache has a 5-minute TTL. Back-to-back operations in
the same session benefit. The commands are written so this happens
naturally; the failure mode to avoid is reading a mutable file
*before* `CLAUDE.md`, which would invalidate the prefix.

### 6.4 `/rewind` vs `/compact` vs `/clear`

Enforced in the skill, not the commands. The meta-rule: **never
compact inside an INGEST.** The wikilink ledger is the "unpredictable
direction" content that compactors summarize away, leaving dangling
links. If context pressure hits mid-ingest, `/clear` with a handoff
brief is always safer.

## 7. Scripts layer

`scripts/` — zero third-party deps for the lexical path.
`sentence-transformers` + `numpy` optional for semantic.

| Script | Input | Output | Invoked by |
|---|---|---|---|
| `build_meta.py` | `wiki/*.md` | `wiki/.meta/*` | every operation, post-pass |
| `build_meta.py --embed-if-enabled` | same | + `embeddings.npy` if enabled | same |
| `build_meta.py --check` | same | drift report to stderr; exit 1 if drift | `tests/test-source-count.sh` |
| `shard_index.py` | `wiki/.meta/*` | `wiki/index.md` + `_index.md` shards + tag shards | `/wiki-compile`, `/wiki-lint`, INGEST post-pass |
| `wiki_search.py <query>` | `wiki/.meta/search-index.tsv` (+ embeddings if present) | ranked paths | `/wiki-search`, `/wiki-query` |
| `semantic.py <query>` | `embeddings.npy` + manifest | ranked paths | `wiki_search.py --mode semantic\|hybrid` |
| `check_extraction.py <path>` | extracted markdown | exit 0/1 + diagnostics | INGEST (pre-write gate) |
| `enable_semantic.sh` | `requirements.txt` | installed deps + initial embed + verification | `/wiki-enable-semantic` |
| `check_invariants.sh` | `tests/*.sh` | per-test PASS/FAIL; exit 1 on any fail | CI, `/wiki-lint` pre-flight |

## 8. Invariant test suite

`tests/*.sh` — 11 tests, shell-only (no pytest / no bash 4 features,
macOS bash 3.2 compatible).

| Test | Invariant |
|---|---|
| `test-frontmatter.sh` | Every page has required YAML fields. |
| `test-wikilinks-resolve.sh` | Every `[[target]]` resolves to a page (any type). |
| `test-outputs-citations-resolve.sh` | Every citation in `wiki/outputs/*.md` resolves — catches QUERY fabrication. |
| `test-unique-slugs.sh` | Non-generated filenames are globally unique. |
| `test-kebab-case.sh` | Filenames are lowercase kebab-case ASCII. |
| `test-source-path-unique.sh` | No two source summaries share a `source_path:`. |
| `test-source-count.sh` | Frontmatter `source_count` matches the backlinks graph. |
| `test-index-size.sh` | `wiki/index.md` stays under the 40K-char (~10K-token) cap. |
| `test-meta-consistent.sh` | `wiki/.meta/*` shape is valid. |
| `test-embeddings-consistent.sh` | Manifest rows = `.npy` rows; vectors L2-normalized; all backlinks slugs covered. |

Tests use plain `grep`, not `rg` — under `bash script.sh` invocation,
`rg` is not resolved (it's a Claude Code shell function). Plain grep
is universal.

## 9. Scale characteristics

Measured (or conservatively estimated) runtime per operation at
various vault sizes. Target hardware: 2021 M1 MacBook Pro.

| Pages | `build_meta.py` | `shard_index.py` | `wiki_search.py` (lexical) | `wiki_search.py` (hybrid) | Check invariants |
|---|---|---|---|---|---|
| 100 | 0.05 s | 0.05 s | 0.05 s | 0.08 s | 0.5 s |
| 1,000 | 0.3 s | 0.3 s | 0.1 s | 0.15 s | 3 s |
| 5,000 | 1.5 s | 1.2 s | 0.3 s | 0.4 s | 15 s |
| 10,000 | 3 s | 2.5 s | 0.5 s | 0.7 s | 30 s |
| 50,000 | 15 s | 12 s | 2.5 s | 3 s (numpy dot on 75MB matrix) | ~2 min |

The ceiling isn't any one of these; it's the point where numpy dot
becomes slow enough to feel interactive-bad (~100ms+ per query),
which happens around 50K–100K pages. Swap point to faiss is
documented in [`DEFERRED.md`](DEFERRED.md).

## 10. Extension points

### 10.1 Adding a new source type

Use `/wiki-new-template`. It scaffolds a new template under
`templates/` and wires it into `CLAUDE.md` INGEST step 2b. Subfolder
under `raw/` gets created on first use.

### 10.2 Adding a new invariant test

1. Create `tests/test-<name>.sh` — must be executable, use plain
   `grep` not `rg`, exit 0 on pass, non-zero on fail.
2. `scripts/check_invariants.sh` picks it up automatically.

### 10.3 Swapping the embedding model

Set `ZWIKI_EMBED_MODEL` before running `build_meta.py --embed`:

```bash
export ZWIKI_EMBED_MODEL="BAAI/bge-small-en-v1.5"
python3 scripts/build_meta.py --embed
```

The manifest stores text SHAs, not vectors, so switching models will
re-embed every page (as expected).

### 10.4 Adding query expansion

`scripts/wiki_search.py` has a reserved slot for query rewriting.
The intended pattern:
1. Run query through a cheap LLM (Claude Haiku) to produce 2
   alternative phrasings.
2. Run hybrid search with each phrasing.
3. RRF-merge all three result lists.

Not implemented — adds a per-query LLM call. See
[`DEFERRED.md`](DEFERRED.md) §Tier 2.

### 10.5 Swapping numpy for faiss

When numpy dot-product becomes slow at scale, swap
`scripts/semantic.py` to use `faiss.IndexFlatIP` (same math, BLAS-
optimized) or `faiss.IndexHNSWFlat` (sub-linear). The `.npy` file
feeds directly into `faiss.deserialize_index`. The embedding
pipeline doesn't change.

## 11. Design decisions — why not X

| Decision | Alternative considered | Why we chose what we chose |
|---|---|---|
| Plain markdown substrate | SQLite, Postgres, vector DB | Renders on GitHub. Portable forever. Git is the backup. `cat wiki/**/*.md` is the worst-case reader. |
| numpy for vectors | faiss | numpy is fast enough through ~50K pages. faiss has CPU-arch-specific index incompatibility (AVX2 indices can't be loaded on generic-CPU machines — documented in faiss-wheels' own README). Swap path documented in `DEFERRED.md`. |
| Sidecar cache, not frontmatter-only | `source_count` in frontmatter is authoritative | Recomputing from frontmatter scans is O(N) per lookup. Sidecar makes it O(1). Frontmatter stays the human-readable source of truth; sidecar is derived and reconciled by LINT. |
| Shell tests, not pytest | pytest with fixtures | Tests must work on a fresh clone with zero install. Shell + `grep` + Python stdlib one-liners hit that bar. |
| Subagent dispatch for batch INGEST | Sequential inline | Parent context stays under context-rot threshold (300–400K tokens) regardless of batch size. Enables 20+-source batches without degradation. |
| User-gated promotion (`/wiki-promote`) | Auto-promote in INGEST post-pass | Promotion generates 300–1200 words per page; a 20-stub batch is ~$4.50 at Opus pricing. Explicit gate makes the cost user-visible. |
| `/wiki-enable-semantic` over auto-install | Bundle numpy + sentence-transformers by default | Core tooling must work with zero pip installs. Semantic is genuinely optional — pure lexical is fine at vault sizes under ~500 sources. |
| Grep-first retrieval | Always load `wiki/index.md` | At 2000 sources, `index.md` would be ~40K tokens per query. Grep-first keeps per-query context bounded by result set size, not vault size. |
